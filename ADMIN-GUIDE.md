## 创建管理员账户

### 首次创建

在项目根目录运行以下命令：

```bash
npm run create-admin
```

按照提示输入：
1. **管理员用户名**（至少3个字符）
2. **管理员邮箱**（有效的邮箱格式）
3. **管理员密码**（至少6个字符）
4. **确认密码**

示例：
```
📌 说明：管理员账户仅用于管理其他用户，不会创建 SillyTavern 实例。

请输入管理员用户名: admin
请输入管理员邮箱: admin@example.com
请输入管理员密码 (至少6个字符): ******
请再次输入密码确认: ******

✅ 管理员账户创建成功！
```

> **重要说明**：管理员账户是纯管理账户，不包含 SillyTavern 实例。如果您既需要管理功能又需要使用 SillyTavern，请创建两个账户：
> - 一个管理员账户用于管理
> - 一个普通用户账户用于使用 SillyTavern

### 创建多个管理员

建议创建至少2个管理员账户作为备份，防止单一管理员账户丢失导致无法管理系统。

重复运行 `npm run create-admin` 命令即可创建多个管理员。

## 登录管理员账户

1. 访问 `http://localhost:3000/login.html`
2. 输入管理员用户名和密码
3. 点击登录

登录成功后：
- 管理员账户会自动跳转到管理员面板（`/admin.html`）
- 不会显示用户控制台，因为管理员账户没有 SillyTavern 实例
- 直接进入系统管理界面

## 管理员面板功能

访问路径：`http://localhost:3000/admin.html`

### 功能概览

管理员面板提供以下核心功能：

#### 1. 系统统计
- 总用户数
- 管理员数量
- 普通用户数量
- 运行中实例数
- 已停止实例数
- 总CPU使用率
- 总内存使用量

#### 2. 用户管理表格
显示所有用户的详细信息：
- 用户名
- 邮箱
- 角色（管理员/用户）
- 端口号
- 实例状态
- SillyTavern版本
- 安装状态
- 注册时间
- 操作按钮

#### 3. 实例监控
实时显示所有正在运行的实例：
- 用户名
- 运行状态
- CPU使用率
- 内存使用量
- 运行时间
- 重启次数

## 用户管理

### 查看用户信息

在用户管理表格中可以看到：
- 所有注册用户的完整信息
- 用户的实例运行状态
- SillyTavern安装状态

### 启动/停止用户实例

**启动实例：**
1. 找到目标用户
2. 点击 **▶️** 按钮
3. 确认操作
4. 等待实例启动（通常1-2秒）

**停止实例：**
1. 找到目标用户
2. 点击 **⏸️** 按钮
3. 确认操作
4. 实例立即停止

**重启实例：**
1. 点击 **🔄** 按钮
2. 确认操作
3. 实例会先停止再启动

### 删除用户

⚠️ **警告：此操作不可恢复！**

删除用户将：
- 停止并删除 PM2 实例
- 删除 SillyTavern 代码目录
- 删除用户数据目录
- 从数据库中删除用户记录

**操作步骤：**
1. 点击用户操作栏的 **🗑️** 按钮
2. 阅读警告信息并确认
3. 输入用户名进行二次确认
4. 点击确定完成删除

**保护机制：**
- 不能删除自己的账户
- 不能删除最后一个管理员

## 实例管理

### 批量操作

管理员可以同时管理多个用户的实例，无需逐个登录用户账户。

### 实时监控

实例监控表格每5秒自动刷新，显示：
- 最新的CPU和内存使用情况
- 准确的运行时间
- 实例状态变化

### 异常处理

如果实例出现异常（如崩溃、无响应）：
1. 在管理员面板查看实例状态
2. 尝试重启实例
3. 如果问题持续，检查用户的错误日志
4. 必要时可以停止实例进行维护

## 角色管理

### 提升用户为管理员

1. 找到目标用户
2. 点击 **👤** 按钮（普通用户图标）
3. 确认操作
4. 用户角色变更为管理员（显示 **👑**）

用户将获得：
- 访问管理员面板的权限
- 管理所有用户和实例的权限
- 系统统计查看权限

### 降级管理员为普通用户

1. 找到目标管理员
2. 点击 **👑** 按钮（管理员图标）
3. 确认操作
4. 用户角色变更为普通用户（显示 **👤**）

**保护机制：**
- 不能取消最后一个管理员的权限
- 系统会自动检查并阻止此操作

## 系统监控

### 系统统计卡片

实时显示系统整体运行状况：

**用户统计：**
- 了解用户增长趋势
- 监控管理员数量
- 评估系统负载

**实例统计：**
- 查看活跃用户数量
- 识别空闲资源
- 优化资源分配

**资源统计：**
- 监控CPU使用率
- 跟踪内存消耗
- 预防资源耗尽

### 自动刷新

所有数据每5秒自动刷新，无需手动操作。

## 安全注意事项

### 管理员账户安全

1. **使用强密码**
   - 至少12个字符
   - 包含大小写字母、数字和特殊字符
   - 不使用常见密码

2. **定期更换密码**
   - 建议每3-6个月更换一次
   - 不重复使用旧密码

3. **创建备份管理员**
   - 至少2个管理员账户
   - 防止单点故障

### 权限管理最佳实践

1. **最小权限原则**
   - 只给需要的人管理员权限
   - 定期审查管理员列表
   - 及时撤销不再需要的权限

2. **操作审计**
   - 记录重要的管理操作
   - 定期检查系统日志
   - 识别异常活动

3. **谨慎删除**
   - 删除用户前务必确认
   - 考虑先停用而非删除
   - 保留必要的备份

### 数据备份

定期备份以下内容：

1. **数据库文件**
   - `database.sqlite`
   - 包含所有用户信息

2. **用户数据目录**
   - `data/` 目录下的所有子目录
   - 包含用户的 SillyTavern 配置

3. **日志文件**
   - `logs/` 目录
   - 用于问题排查

## 常见问题

### Q: 管理员账户可以使用 SillyTavern 吗？

A: 不可以。管理员账户是纯管理账户，不包含 SillyTavern 实例。如果您需要同时管理和使用 SillyTavern，请创建两个账户：一个管理员账户用于管理，一个普通用户账户用于使用 SillyTavern。

### Q: 忘记管理员密码怎么办？

A: 需要直接修改数据库或使用其他管理员账户来重置密码。建议始终保持至少2个管理员账户。

### Q: 可以将管理员降级为普通用户吗？

A: 可以，但至少需要保留一个管理员账户。系统会阻止删除或降级最后一个管理员。

### Q: 删除用户会影响其他用户吗？

A: 不会。每个用户的数据完全隔离，删除一个用户不会影响其他用户。

### Q: 管理员可以看到用户的 SillyTavern 数据吗？

A: 管理员可以访问用户的数据目录，但建议尊重用户隐私，仅在必要时（如故障排查）访问。

### Q: 如何监控系统资源使用？

A: 在管理员面板的系统统计卡片中可以看到总CPU和内存使用量。如需更详细的监控，建议使用专业的系统监控工具。

## 技术支持

如遇到问题，请检查：
1. 系统日志（`logs/` 目录）
2. PM2 进程状态（`pm2 list`）
3. 数据库完整性
4. 磁盘空间是否充足
