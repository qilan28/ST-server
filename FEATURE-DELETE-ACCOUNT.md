# 🗑️ 用户删除账号功能

## 功能概述

在用户面板（Dashboard）中添加了删除账号的功能，允许用户自行删除自己的账号及所有相关数据。

---

## ✨ 功能特性

### 1. 完整的删除流程

删除账号时会执行以下操作：
- ✅ 停止并删除 PM2 实例
- ✅ 删除 SillyTavern 程序目录
- ✅ 删除用户数据目录（包括对话记录、角色、设置等）
- ✅ 从数据库中删除用户记录
- ✅ 自动重新生成 Nginx 配置并重载
- ✅ 清除浏览器 Cookie 和本地存储

### 2. 双重安全确认

为防止误操作，设置了两层确认机制：
1. **第一层确认**：要求用户输入 `DELETE` 来确认删除意图
2. **第二层确认**：再次弹窗确认是否真的要删除

### 3. 管理员保护

- ❌ 不能删除最后一个管理员账号
- ✅ 如果是管理员且不是最后一个，可以删除
- ✅ 普通用户可以随时删除自己的账号

### 4. 友好的用户界面

- 🎨 专门的"危险操作"区域（红色边框）
- ⚠️ 清晰的警告提示
- 📝 详细的操作说明
- 🔒 醒目的红色删除按钮

## 📋 技术实现

### 后端 API（`routes/auth.js`）

```javascript
router.delete('/account', authenticateToken, async (req, res) => {
    // 1. 验证用户身份
    // 2. 防止删除最后一个管理员
    // 3. 停止并删除 PM2 实例
    // 4. 删除 SillyTavern 目录
    // 5. 删除用户数据目录
    // 6. 从数据库删除用户
    // 7. 重新生成 Nginx 配置
    // 8. 清除 Cookie
    // 9. 返回成功响应
```

**API 端点**：
- **URL**: `DELETE /api/auth/account`

**CSS**（`public/css/style.css`）：

## 🚀 使用流程

### 用户操作流程

1. **访问用户面板**
   - 登录后进入 Dashboard
   - 滚动到页面底部

2. **点击删除按钮**
   - 找到"危险操作"区域
   - 点击"🗑️ 删除我的账号"

3. **第一次确认**
   - 阅读警告信息
   - 输入 `DELETE` 确认
   - 点击"确定"

4. **第二次确认**
   - 再次阅读警告
   - 点击"确定"最终确认

5. **自动删除**
   - 显示"⏳ 删除中..."
   - 执行删除操作
   - 显示成功消息
   - 自动跳转到首页

### 系统执行流程

```
用户点击删除
    ↓
第一次确认（输入 DELETE）
    ↓
第二次确认（最后确认）
    ↓
调用 API: DELETE /api/auth/account
    ↓
【后端处理】
    ├─ 验证用户身份
    ├─ 检查是否为最后一个管理员
    ├─ 停止 PM2 实例
    ├─ 删除 SillyTavern 目录
    ├─ 删除用户数据目录
    ├─ 删除数据库记录
    ├─ 重新生成 Nginx 配置
    └─ 清除 Cookie
    ↓
【前端处理】
    ├─ 清除本地存储
    ├─ 显示成功消息
    └─ 跳转到首页
```

## 🛡️ 安全措施

### 1. 身份验证

- ✅ 使用 `authenticateToken` 中间件验证身份
- ✅ 只能删除自己的账号，不能删除他人账号

### 2. 管理员保护

```javascript
if (user.role === 'admin') {
    const adminCount = getAllUsers().filter(u => u.role === 'admin').length;
    if (adminCount <= 1) {
        return res.status(400).json({ 
            error: '不能删除最后一个管理员用户'
        });
    }
}
```

### 3. 双重确认

- ✅ 第一次确认：输入 `DELETE`
- ✅ 第二次确认：再次点击确定
- ✅ 防止误点击导致删除

### 4. 数据清理

- ✅ 完整删除所有用户相关数据
- ✅ 防止数据泄露和残留

---

## ⚠️ 注意事项

### 对于用户

1. **数据不可恢复**
   - 删除后所有数据永久丢失
   - 包括对话记录、角色、设置等
   - 请在删除前备份重要数据

2. **实例自动停止**
   - 删除账号时会自动停止正在运行的实例
   - 无需手动停止

3. **立即生效**
   - 删除操作立即执行
   - 账号被删除后无法再登录

### 对于管理员

1. **最后一个管理员**
   - 系统会阻止删除最后一个管理员账号
   - 确保系统始终有管理员可以管理

2. **Nginx 配置**
   - 删除用户后会自动重新生成 Nginx 配置
   - 删除的用户路径会从 Nginx 中移除

3. **磁盘空间**
   - 删除用户会释放其占用的磁盘空间
   - 包括 SillyTavern 程序和用户数据

## 🔄 回滚方案

如果用户误删除账号，只能通过以下方式恢复：

1. **重新注册**
   - 使用相同的用户名注册新账号
   - 会分配新的端口
   - 无法恢复之前的数据

2. **数据备份恢复（如果有备份）**
   - 管理员手动恢复数据库记录
   - 恢复用户数据目录
   - 恢复 SillyTavern 目录
   - 重新生成 Nginx 配置

**建议**：
- 在删除前提醒用户备份重要数据
- 可以考虑添加"软删除"功能（标记为删除但保留数据一段时间）
