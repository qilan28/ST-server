# SillyTavern 转发多实例配置指南

本文档说明如何配置和使用SillyTavern多开管理平台的多实例转发功能。

## 功能介绍

多实例转发功能允许管理员在后台配置多个SillyTavern实例的地址，然后通过统一的URL规则提供给用户访问。系统的工作流程如下：

1. 所有用户都通过 `http://域名:端口/用户名/st/` 访问他们的SillyTavern实例
2. 访问请求会被转发到主转发服务器（默认在7091端口）
3. 主转发服务器会从配置的实例地址中随机选择一个，并将请求转发到该实例

## 配置步骤

### 1. 启用Nginx模式

首先需要在管理员面板中启用Nginx模式：

1. 登录管理员账户，进入管理员面板
2. 在"系统配置"中找到"Nginx反向代理配置"部分
3. 勾选"启用Nginx模式"
4. 填写域名和端口
5. 点击"保存配置"，然后点击"生成配置文件"

### 2. 配置实例地址

在管理员面板的"实例配置管理"部分：

1. 设置主转发端口（默认是7091），这是内部转发服务器监听的端口
2. 添加多个SillyTavern实例地址，例如：`http://8.210.210.211:7092`
3. 可以添加多个实例地址，系统会自动进行负载均衡

### 3. 启动转发服务器

有两种方式启动转发服务器：

#### 方式1：单独启动

```bash
# 在命令行中执行
cd ST-server
npm run start-forward
```

或者直接双击 `scripts/start-forward-server.bat`

#### 方式2：使用PM2管理（推荐）

```bash
# 在命令行中执行
cd ST-server
npx pm2 start ecosystem.config.js
npx pm2 save
```

或者直接双击 `scripts/start-all-pm2.bat`

### 4. 应用Nginx配置

在Linux服务器上，应用新的Nginx配置：

```bash
# 在命令行中执行
cd ST-server
npm run deploy-nginx
```

对于Windows系统，可以使用自己的Nginx服务器，将生成的配置文件复制到Nginx配置目录。

## 访问方式

配置完成后，用户可以通过以下URL访问他们的SillyTavern实例：

```
http://域名:端口/用户名/st/
```

例如：`http://example.com:80/user123/st/`

## 故障排查

1. **无法访问实例**：检查转发服务器是否已启动，查看日志 `npx pm2 logs st-forward`
2. **转发错误**：确保添加的实例地址可以正常访问
3. **Nginx配置错误**：检查Nginx错误日志，尝试重新生成和部署配置文件

## 注意事项

- 主转发服务器需要保持运行，否则用户将无法访问实例
- 添加的实例地址需要确保从转发服务器所在的机器可以访问
- 如果修改了端口或实例配置，需要重新生成并应用Nginx配置
