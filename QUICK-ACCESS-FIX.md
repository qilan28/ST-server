# ✅ 快速修复：访问地址已更新为直接端口模式

## 🐛 原问题

通过 `http://119.8.118.149:7092/123/st/` 访问时，所有静态资源都返回 404。

**原因：** SillyTavern 不支持在子路径下运行。

## ✅ 已修复

**修改后的访问方式：直接端口访问**

```
管理平台: http://119.8.118.149:3000/
用户 123:  http://119.8.118.149:3001/
用户 111:  （管理员，无需访问）
```

---

## 🚀 立即使用

### 步骤 1：重启服务器

```bash
cd /root/ST-server

# 重启服务器以应用新的 URL 生成逻辑
pm2 restart all
# 或
npm start
```

### 步骤 2：开放端口

```bash
# 开放管理平台端口
sudo ufw allow 3000/tcp

# 开放用户实例端口范围
sudo ufw allow 3001:4000/tcp

# 查看状态
sudo ufw status
```

### 步骤 3：访问实例

**用户 123 访问：**
```
直接打开浏览器访问：http://119.8.118.149:3001/
```

**管理员访问：**
```
管理面板：http://119.8.118.149:3000/admin.html
```

## ❓ 常见问题

### Q: Nginx 配置还有用吗？

**A:** 目前不使用 Nginx 路径转发了。如果您想使用 Nginx，可以：
- 配置子域名转发（推荐）
- 使用 Nginx 作为统一入口，但仍代理到不同端口

### Q: URL 包含端口号不美观？

**A:** 是的，但这是最兼容的方式。如果想要美观的 URL：
- 短期：可以接受端口号
- 长期：配置域名使用子域名方案

### Q: 安全性如何？

**A:** 
- ✅ 每个用户只能访问自己的实例
- ✅ JWT 认证保护管理平台
- ✅ 可以配置防火墙规则限制访问
- ✅ 建议：生产环境配置 HTTPS

### Q: 可以改回路径转发吗？

**A:** 技术上可以，但不推荐，因为：
- ❌ SillyTavern 不支持子路径
- ❌ 会导致资源 404
- ❌ 需要复杂的 Nginx 重写规则
- ❌ 可能影响某些功能
