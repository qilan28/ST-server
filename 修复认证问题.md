# 🔧 修复认证问题 - Cookie 无法设置

## 问题原因

1. **认证中间件只检查 Authorization header，没有检查 Cookie** ❌
2. **Cookie 设置了 `secure: true`，但你用的是 HTTP（不是 HTTPS）** ❌

浏览器会拒绝在 HTTP 下设置 secure cookie，导致登录后没有 `st_token`。

---

## ✅ 已修复

### 1. 修改了 `middleware/auth.js`
现在会优先从 Cookie 读取 `st_token`：

```javascript
let token = req.cookies?.st_token;  // 优先从 Cookie 获取

if (!token) {
    const authHeader = req.headers['authorization'];
    token = authHeader && authHeader.split(' ')[1];  // 其次从 header 获取
}
```

### 2. 修改了 `routes/auth.js`
Cookie 设置改为支持 HTTP：

```javascript
res.cookie('st_token', token, {
    httpOnly: true,
    secure: false,  // ✅ 改为 false，支持 HTTP
    maxAge: 24 * 60 * 60 * 1000,
    sameSite: 'lax',
    path: '/'  // ✅ 确保整个网站都能访问
});
```

## 💡 为什么会出现这个问题？

1. **HTTP vs HTTPS**：
   - 原代码：`secure: process.env.NODE_ENV === 'production'`
   - 如果设置了生产环境，但用 HTTP 访问，浏览器拒绝设置 secure cookie

2. **认证方式不一致**：
   - 登录设置的是 Cookie
   - 认证中间件只检查 Authorization header
   - 导致有 Cookie 但验证失败
