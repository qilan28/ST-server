# 修复说明：用户控制台访问地址跳转问题

## 问题描述
用户在控制台点击"访问地址"链接时，会被重定向到 `access-denied.html` 页面，无法正常访问 SillyTavern 实例。

## 问题原因
1. **Cookie 传递问题**：用户登录后，虽然 localStorage 中保存了 token，但在某些情况下（如清除了 Cookie、新标签页等），`st_token` Cookie 可能丢失
2. **Nginx 权限验证**：Nginx 使用 `auth_request` 机制验证用户权限，需要从 Cookie 中读取 `st_token`
3. **缺少自动修复**：当 Cookie 丢失时，系统没有自动重新设置 Cookie 的机制

## 修复方案

### 1. 优化 dashboard.js 访问地址点击处理
**文件**: `ST-server/public/js/dashboard.js`

**修改内容**:
- 阻止访问地址链接的默认行为
- 在打开新标签页前检查 `st_token` Cookie 是否存在
- 如果 Cookie 不存在但 localStorage 有 token，自动重新设置 Cookie
- 添加错误提示，帮助用户排查问题
- 确认 Cookie 设置成功后才打开新标签页

**代码逻辑**:
```javascript
accessLink.onclick = function(e) {
    e.preventDefault(); // 阻止默认行为
    
    // 检查 Cookie
    if (!cookies.includes('st_token') && token) {
        // 重新设置 Cookie
        setCookie('st_token', token);
    }
    
    // 打开新标签页
    window.open(accessUrl, '_blank');
};
```

### 2. 增强 access-denied.html 智能处理
**文件**: `ST-server/public/access-denied.html`

**修改内容**:
- 添加智能诊断功能，自动检测问题原因
- 如果检测到是 Cookie 问题，自动设置 Cookie
- 3 秒后自动重定向到目标实例页面
- 修改按钮文案，提供更友好的操作选项

**智能诊断逻辑**:
```javascript
function tryAutoFix() {
    const token = localStorage.getItem('token');
    const currentUsername = localStorage.getItem('username');
    
    // 如果用户已登录且访问的是自己的实例
    if (token && currentUsername === targetUsername) {
        // 检查并设置 Cookie
        if (!document.cookie.includes('st_token')) {
            setCookie('st_token', token);
            // 3秒后自动重定向
            setTimeout(() => {
                window.location.href = `/${targetUsername}/st/`;
            }, 3000);
        }
    }
}
```

## 技术细节

### Cookie 设置策略
```javascript
function setCookie(name, value, days = 365) {
    document.cookie = `${name}=${value}; path=/; max-age=${days * 24 * 60 * 60}; SameSite=Lax`;
    return document.cookie.includes(`${name}=`);
}
```

**参数说明**:
- `path=/`: Cookie 在整个网站有效
- `max-age=365天`: Cookie 有效期 1 年
- `SameSite=Lax`: 防止 CSRF 攻击，同时允许顶级导航携带 Cookie

### Nginx 权限验证流程
1. 用户访问 `/123/st/`
2. Nginx 触发 `auth_request /auth-check/123`
3. 后端验证 Cookie 中的 `st_token`
4. 验证通过：返回 200，正常访问
5. 验证失败：返回 401/403，重定向到 `access-denied.html`

3. 如果用户手动清除了 localStorage，需要重新登录
