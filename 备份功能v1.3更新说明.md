# 🔄 备份功能 v1.3 更新说明

## ✨ 新增功能

### 1. 邮箱配置支持 ✅
现在需要配置 Hugging Face 邮箱，用于 Git commit 身份验证。

**数据库更新**:
- 添加 `hf_email` 字段到 users 表
- 自动迁移，无需手动操作

**界面更新**:
- 配置页面新增"邮箱"输入框
- 必填字段：Token、仓库名、邮箱

### 2. 文件命名优化 ✅
**旧格式**:
```
123_backup_2025-12-16_18-30-45.zip
```

**新格式** (纯时间戳):
```
1734432645123.zip
```

**优势**:
- 文件名更简洁
- 自动排序（按时间）
- 避免特殊字符问题
- 唯一性更强（毫秒级）

### 3. Commit 信息优化 ✅
**旧格式**:
```
Backup: 123_backup_2025-12-16_18-30-45.zip (15.50 MB)
```

**新格式**:
```
123-2025-12-16 18:30:45 (15.50 MB)
```

**优势**:
- 一眼看出谁备份的
- 清晰的时间戳
- 文件大小一目了然
- 更易读和搜索

### 4. Git 用户信息 ✅
使用用户配置的邮箱进行 Git commit:
```javascript
git config user.email "user@example.com"
git config user.name "123"  // 使用用户名作为 Git name
```

## 📋 配置步骤

### 1. 填写备份配置
登录控制台 → 数据备份模块，填写：

- **Hugging Face Token**: `hf_xxxxxxxxxxxxx`
- **仓库名**: `username/repo-name`
- **邮箱**: `your-email@example.com` ⭐ **新增**

### 2. 测试连接
点击"测试连接"验证配置是否正确。

### 3. 执行备份
点击"立即备份"开始备份。

## 🔄 迁移说明

### 自动迁移
重启服务后自动执行：
```bash
pm2 restart st-server
```

日志会显示：
```
[Database] 添加 hf_email 字段...
[Database] ✅ hf_email 字段添加成功
```

### 旧配置兼容
- 如果没有配置邮箱，默认使用 `backup@sillytavern.local`
- 旧的备份文件不受影响
- 新备份将使用新格式

## ⚠️ 注意事项

1. **首次使用需配置邮箱**
   - 登录后在备份模块添加邮箱
   - 邮箱格式：`xxx@xxx.xxx`

2. **文件名为时间戳**
   - 毫秒级 Unix 时间戳
   - 可用工具转换为可读时间

3. **Commit 信息包含完整信息**
   - 用户名
   - 备份时间（精确到秒）
   - 文件大小

4. **兼容性**
   - 旧版本备份文件不受影响
   - 可同时存在新旧格式文件

## 🚀 使用建议

### 查看备份历史
访问 Hugging Face 仓库，按时间倒序查看：
```
https://huggingface.co/datasets/{username}/{repo}/tree/main
```

### 恢复特定备份
1. 找到对应时间戳的文件
2. 下载并解压
3. 恢复到 `st-data` 目录

### 清理旧备份
定期删除过旧的备份文件，节省空间。

## 📞 技术支持

2. 确认 HF Token 有 Write 权限
3. 查看服务器日志：`pm2 logs st-server`
