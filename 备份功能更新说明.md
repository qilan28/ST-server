# 🔄 备份功能更新说明 v1.2.0

## ⚠️ 重要更新

Hugging Face 已弃用旧的上传 API，我们已更新为使用 **Git 方式**上传，更稳定可靠！

## 📦 系统要求

确保服务器已安装：

```bash
# 检查 Git
git --version

# 检查 Git LFS（可选，用于大文件）
git lfs version

# 如果没有 Git LFS，安装它（推荐）
# Ubuntu/Debian
sudo apt-get install git-lfs
git lfs install

# CentOS/RHEL
sudo yum install git-lfs
git lfs install
```

**注意**：不需要安装额外的 npm 依赖！

## ✨ 新功能

### 1. Git 方式上传（更可靠）
- ✅ 使用 Git 克隆、提交、推送
- ✅ 自动检测并使用 Git LFS（文件 > 10MB）
- ✅ 支持任意大小文件（理论上无限制）
- ✅ 断点续传支持（Git 协议特性）

### 2. 智能分支检测
- 自动尝试 `main` 和 `master` 分支
- 支持空仓库初始化

### 3. 更好的错误处理
- Token 认证失败
- 仓库不存在
- 网络超时
- Git 操作失败

## 🔧 使用方法

### 方式一：Web 界面（推荐）
1. 登录控制台
2. 进入 **💾 数据备份** 模块
3. 配置 HF Token 和仓库
4. 点击 **☁️ 立即备份**

### 方式二：命令行测试
```bash
cd /root/ST-server
node --input-type=module -e "
import('./utils/hf-backup.js').then(m => {
  m.backupToHuggingFace(
    '/root/ST-server/data/123/st-data',
    '123',
    'hf_xxxxxxxxxxxx',
    'username/repo-name'
  ).then(r => console.log('成功:', r))
   .catch(e => console.error('失败:', e));
});
"
```

## 📋 大文件备份建议

如果你的数据超过 5GB，建议：

### 选项 1: 清理缓存
```bash
cd /root/ST-server/data/{username}/st-data
rm -rf _cache/*
rm -rf _uploads/*
```

### 选项 2: 使用 Git LFS
详见 `大文件备份说明.md`

### 选项 3: 分批备份
只备份重要数据（characters、chats）

## 🐛 故障排查

### 错误: "git: command not found"
**原因**: 系统未安装 Git  
**解决**: 
```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install git

# CentOS/RHEL
sudo yum install git
```

### 错误: "Authentication failed"
**原因**: Token 无效或权限不足  
**解决**: 
1. 检查 HF Token 是否正确
2. 确认 Token 有 Write 权限
3. 确认仓库属于该账号

### 错误: 上传超时
**原因**: 文件太大或网络不稳定  
**解决**: 
1. 检查文件大小
2. 清理不必要的数据
3. 使用 Git LFS 手动上传

### 错误: 内存不足
**原因**: 文件太大，压缩时内存不足  
**解决**: 
```bash
# 增加 Node.js 内存限制
NODE_OPTIONS="--max-old-space-size=4096" pm2 start server.js
```

## 📊 性能参考

基于不同文件大小的预估时间（网络速度: 10 Mbps）：

| 文件大小 | 压缩时间 | 上传时间 | 总时间 |
|---------|---------|---------|--------|
| 100 MB  | 10-30秒 | 1-2分钟 | ~2分钟 |
| 500 MB  | 30-60秒 | 5-8分钟 | ~8分钟 |
| 1 GB    | 1-2分钟 | 10-15分钟 | ~17分钟 |
| 5 GB    | 5-10分钟 | 50-80分钟 | ~90分钟 |
| 10 GB   | 10-20分钟 | 100-160分钟 | ~180分钟 |

**建议**: 超过 5GB 的文件不建议通过 Web 界面上传
