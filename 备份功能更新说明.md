# 🔄 备份功能更新说明 v1.1.0

## ⚠️ 重要更新

Hugging Face 已弃用旧的上传 API，我们已更新为使用官方 Hugging Face Hub SDK。

## 📦 安装新依赖

在服务器上运行以下命令：

```bash
cd /root/ST-server
npm install
```

这将安装以下新依赖：
- `@huggingface/hub@^0.18.0` - 官方 Hugging Face Hub SDK
- `form-data@^4.0.0` - 表单数据处理

## ✨ 新功能

### 1. 支持大文件上传
- ✅ 自动使用 Git LFS 处理大文件
- ✅ 流式上传，减少内存占用
- ✅ 支持 10GB+ 文件（如果网络稳定）

### 2. 文件大小提示
上传时会显示文件大小警告：
- **< 1 GB**: 快速上传
- **1-5 GB**: 需要较长时间
- **> 5 GB**: 建议使用其他方式

### 3. 更好的错误提示
- Token 无效
- 仓库不存在
- 权限不足
- 文件过大

## 🔧 使用方法

### 方式一：Web 界面（推荐）
1. 登录控制台
2. 进入 **💾 数据备份** 模块
3. 配置 HF Token 和仓库
4. 点击 **☁️ 立即备份**

### 方式二：命令行测试
```bash
cd /root/ST-server
node --input-type=module -e "
import('./utils/hf-backup.js').then(m => {
  m.backupToHuggingFace(
    '/root/ST-server/data/123/st-data',
    '123',
    'hf_xxxxxxxxxxxx',
    'username/repo-name'
  ).then(r => console.log('成功:', r))
   .catch(e => console.error('失败:', e));
});
"
```

## 📋 大文件备份建议

如果你的数据超过 5GB，建议：

### 选项 1: 清理缓存
```bash
cd /root/ST-server/data/{username}/st-data
rm -rf _cache/*
rm -rf _uploads/*
```

### 选项 2: 使用 Git LFS
详见 `大文件备份说明.md`

### 选项 3: 分批备份
只备份重要数据（characters、chats）

## 🐛 故障排查

### 错误: "已弃用的端点"
**原因**: 使用了旧版本代码  
**解决**: 
```bash
git pull
npm install
pm2 restart all
```

### 错误: "Cannot find module '@huggingface/hub'"
**原因**: 未安装新依赖  
**解决**: 
```bash
npm install
```

### 错误: 上传超时
**原因**: 文件太大或网络不稳定  
**解决**: 
1. 检查文件大小
2. 清理不必要的数据
3. 使用 Git LFS 手动上传

### 错误: 内存不足
**原因**: 文件太大，压缩时内存不足  
**解决**: 
```bash
# 增加 Node.js 内存限制
NODE_OPTIONS="--max-old-space-size=4096" pm2 start server.js
```

## 📊 性能参考

基于不同文件大小的预估时间（网络速度: 10 Mbps）：

| 文件大小 | 压缩时间 | 上传时间 | 总时间 |
|---------|---------|---------|--------|
| 100 MB  | 10-30秒 | 1-2分钟 | ~2分钟 |
| 500 MB  | 30-60秒 | 5-8分钟 | ~8分钟 |
| 1 GB    | 1-2分钟 | 10-15分钟 | ~17分钟 |
| 5 GB    | 5-10分钟 | 50-80分钟 | ~90分钟 |
| 10 GB   | 10-20分钟 | 100-160分钟 | ~180分钟 |

**建议**: 超过 5GB 的文件不建议通过 Web 界面上传
