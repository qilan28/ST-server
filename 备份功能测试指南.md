# 📋 Hugging Face 备份功能测试指南

## 🔧 安装步骤

### 1. 安装依赖
```bash
cd e:\酒馆\公益云酒馆多开管理平台\ST-server
npm install
```

这会自动安装 `archiver` 包（已添加到 package.json）

### 2. 重启服务器
```bash
npm start
```

## 🧪 测试步骤

### 准备工作

#### 1. 准备 Hugging Face 账号
- 访问 https://huggingface.co 注册账号（如果还没有）
- 登录后访问 https://huggingface.co/settings/tokens
- 创建新 Token，权限选择 **Write**
- 保存 Token（格式：`hf_xxxxxxxxxx`）

#### 2. 创建 Dataset 仓库
- 访问 https://huggingface.co/new-dataset
- 填写仓库名（例如：`sillytavern-backup`）
- 选择 Private 或 Public
- 创建仓库
- 记录完整仓库名（例如：`yourname/sillytavern-backup`）

### 功能测试

#### 测试 1: 保存配置
1. 登录普通用户账号（非管理员）
2. 在控制台找到 **💾 数据备份** 卡片
3. 填写：
   - **Hugging Face 仓库名**: `yourname/sillytavern-backup`
   - **Hugging Face Token**: 粘贴之前创建的 Token
4. 点击 **💾 保存配置**
5. **预期结果**: 显示"✅ 配置保存成功！"

#### 测试 2: 测试连接
1. 在保存配置后
2. 点击 **🔌 测试连接** 按钮
3. **预期结果**: 
   - 显示"✅ 连接成功！"
   - 显示仓库信息（仓库名、作者、类型）

#### 测试 3: 执行备份
1. 确保用户已经安装了 SillyTavern（有 st-data 目录）
2. 点击 **☁️ 立即备份** 按钮
3. 确认备份对话框
4. 等待备份完成（时间取决于数据大小）
5. **预期结果**:
   - 显示"✅ 备份成功！"
   - 显示备份详情：
     - 备份文件名
     - 文件大小
     - 备份时间
     - 下载地址

#### 测试 4: 验证备份文件
1. 点击备份成功后显示的下载地址
2. 访问 Hugging Face 仓库页面
3. **预期结果**:
   - 可以看到上传的 ZIP 文件
   - 文件名格式：`{用户名}_backup_{日期}_{时间}.zip`
   - 可以下载文件

#### 测试 5: 错误处理
1. **测试错误的仓库名**:
   - 输入错误格式的仓库名（例如：`onlyname`，没有 `/`）
   - **预期**: 显示错误提示
   
2. **测试无效 Token**:
   - 输入错误的 Token
   - 点击测试连接
   - **预期**: 显示连接失败

3. **测试不存在的仓库**:
   - 输入不存在的仓库名
   - **预期**: 显示仓库不存在

## 🔍 调试技巧

### 查看服务器日志
备份过程会输出详细日志：
```
[HF Backup] 开始备份流程...
[HF Backup] 数据目录: /path/to/data
[HF Backup] 正在压缩数据目录...
[HF Backup] 压缩完成: xxxx 字节
[HF Backup] 压缩包大小: xx.xx MB
[HF Backup] 正在上传到 Hugging Face...
[HF Backup] 开始上传文件: filename.zip (xx.xx MB)
[HF Backup] ✅ 上传成功
[HF Backup] 清理临时文件...
[HF Backup] ✅ 备份完成！
```

### 常见问题

1. **压缩失败**
   - 检查 data 目录是否存在
   - 检查磁盘空间是否充足

2. **上传失败**
   - 检查网络连接
   - 验证 Token 权限
   - 确认仓库名正确

3. **前端无法加载**
   - 检查浏览器控制台
   - 确认 JavaScript 没有语法错误
