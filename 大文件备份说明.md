# 大文件备份说明

## 📊 文件大小限制

### 自动备份支持范围
- ✅ **< 1 GB**: 快速上传，几分钟内完成
- ⚠️ **1-5 GB**: 需要较长时间（5-15分钟），建议使用稳定网络
- ❌ **> 5 GB**: 上传时间很长（15分钟以上），可能会超时

### Hugging Face 限制
- **单文件大小**: 官方推荐使用 Git LFS 处理 > 10MB 的文件
- **总仓库大小**: 免费账户无限制，但建议合理使用
- **上传超时**: 超大文件可能因网络问题导致上传失败

## 🚀 大文件备份建议方案

### 方案一：压缩优化（推荐）
在备份前清理不必要的数据：

```bash
# 进入数据目录
cd /root/ST-server/data/{username}/st-data

# 删除缓存文件
rm -rf _cache/*
rm -rf _uploads/*

# 删除临时文件
find . -name "*.tmp" -delete
find . -name "*.log" -delete
```

### 方案二：使用 Git LFS 手动上传（超大文件）

#### 1. 安装 Git LFS
```bash
# Ubuntu/Debian
sudo apt-get install git-lfs

# CentOS/RHEL
sudo yum install git-lfs

# macOS
brew install git-lfs
```

#### 2. 初始化 Git LFS
```bash
git lfs install
```

#### 3. 克隆 Hugging Face 仓库
```bash
# 替换为你的仓库名
git clone https://huggingface.co/datasets/{username}/{repo-name}
cd {repo-name}
```

#### 4. 配置 LFS 追踪大文件
```bash
# 追踪所有 .zip 文件
git lfs track "*.zip"
git add .gitattributes
```

#### 5. 上传备份文件
```bash
# 复制备份文件到仓库
cp /root/ST-server/temp/{backup-file}.zip .

# 提交和推送
git add {backup-file}.zip
git commit -m "Backup: {backup-file}.zip"
git push
```

### 方案三：分片备份

对于超大数据目录，可以分批备份：

#### 1. 按目录分批备份
```bash
# 只备份对话记录
tar -czf chats_backup.tar.gz st-data/chats/

# 只备份角色文件
tar -czf characters_backup.tar.gz st-data/characters/

# 只备份设置
tar -czf settings_backup.tar.gz st-data/default-user/
```

#### 2. 分别上传到不同文件
这样可以避免单个文件过大。

### 方案四：使用第三方存储

如果 Hugging Face 不适合，可以考虑：
- **Google Drive**: 15GB 免费空间
- **OneDrive**: 5GB 免费空间
- **Dropbox**: 2GB 免费空间
- **阿里云OSS/腾讯云COS**: 按量付费

## 📝 备份最佳实践

### 定期清理
```bash
# 删除旧的对话记录（保留最近30天）
find st-data/chats/ -type f -mtime +30 -delete

# 清理缓存
rm -rf st-data/_cache/*
rm -rf st-data/_uploads/*
```

### 选择性备份
只备份重要数据：
- ✅ `characters/` - 角色定义
- ✅ `chats/` - 对话记录
- ✅ `settings.json` - 设置
- ❌ `_cache/` - 缓存（可重建）
- ❌ `_uploads/` - 临时上传（可删除）

### 压缩率优化
```bash
# 使用最高压缩率（速度慢但文件小）
zip -9 -r backup.zip st-data/

# 排除缓存和临时文件
zip -r backup.zip st-data/ -x "st-data/_cache/*" "st-data/_uploads/*"
```

## ⚠️ 故障排查

### 上传失败
1. **检查网络**: 确保网络稳定
2. **检查 Token**: 验证 HF Token 是否有效
3. **检查仓库权限**: 确认有写入权限
4. **尝试小文件**: 先测试小文件上传

### 超时问题
```bash
# 手动压缩测试
cd /root/ST-server
node -e "
import('./utils/hf-backup.js').then(m => {
  m.compressDirectory(
    '/root/ST-server/data/{username}/st-data',
    '/root/ST-server/temp/test.zip'
  ).then(() => console.log('压缩完成'));
});
"
```

### 内存不足
如果备份时出现内存错误：
```bash
# 增加 Node.js 内存限制
NODE_OPTIONS="--max-old-space-size=4096" npm start
```

## 💡 提示

1. **定期备份**: 建议每周备份一次重要数据
2. **多重备份**: 不要只依赖一个备份，建议使用多个存储位置
3. **测试恢复**: 定期测试备份文件是否可以正常恢复
4. **文档记录**: 记录备份的内容和时间
