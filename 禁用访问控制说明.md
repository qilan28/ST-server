# 禁用访问控制说明

## 📌 什么是访问控制？

访问控制是 Nginx 的一个安全功能，确保：
- ✅ 每个用户只能访问自己的 SillyTavern 实例
- ✅ 需要登录才能访问实例
- ✅ 防止未授权访问

## ⚠️ 禁用后的影响

禁用访问控制后：
- ❌ **任何人都能访问所有用户的实例**（无需登录）
- ❌ 实例 URL 泄露后，任何人都能使用
- ❌ 没有访问日志和权限控制
- ⚠️ **不推荐在公网环境使用**

## 🔧 如何禁用访问控制

### 方法 1：使用脚本（推荐）

```bash
# 1. 禁用访问控制
npm run disable-access-control

# 2. 重新生成 Nginx 配置
npm run generate-nginx

# 3. 重载 Nginx
sudo nginx -s reload
```

### 方法 2：手动修改配置

编辑 `config.json` 文件：

```json
{
  "nginx": {
    "enabled": true,
    "domain": "your-domain.com",
    "port": 80,
    "enableAccessControl": false  // 改为 false
  }
}
```

然后重新生成并重载 Nginx：

```bash
npm run generate-nginx
sudo nginx -s reload
```

## 🔄 如何重新启用访问控制

```bash
# 1. 启用访问控制
npm run enable-access-control

# 2. 这个脚本会自动完成：
#    - 修改配置
#    - 重新生成 Nginx 配置
#    - 重载 Nginx
```

## 📝 验证是否生效

### 检查配置文件

```bash
npm run check-config
```

输出应该显示：
```
Nginx 配置:
  enableAccessControl: ⚪ 禁用
```

### 测试访问

1. **禁用访问控制后**：
   - 打开浏览器，访问任意实例 URL（如 `http://your-domain/123/st/`）
   - 应该能直接访问，无需登录

2. **启用访问控制后**：
   - 访问实例 URL 时会显示"访问被拒绝"
   - 需要登录后才能访问自己的实例

## 🎯 适用场景

### 适合禁用访问控制的情况：
- ✅ 内网环境，只有信任的用户能访问
- ✅ 单用户使用
- ✅ 调试和测试
- ✅ 通过其他方式（如防火墙、VPN）控制访问

### 不适合禁用的情况：
- ❌ 公网环境
- ❌ 多用户环境
- ❌ 需要访问控制和审计
- ❌ 数据敏感的场景

## 🔍 Nginx 配置对比

### 启用访问控制时

```nginx
location /123/st/ {
    # 🔒 访问控制：只有 123 用户才能访问
    auth_request /auth-check-internal/123;
    error_page 401 403 = @access_denied;
    
    # ... 其他配置
}
```

### 禁用访问控制时

```nginx
location /123/st/ {
    # 无访问控制，任何人都能访问
    
    # ... 其他配置
}
```

## 💡 安全建议

如果禁用了访问控制，建议：

1. **使用强密码** - SillyTavern 本身可能有登录功能
2. **防火墙限制** - 只允许特定 IP 访问
3. **VPN 访问** - 通过 VPN 连接后才能访问
4. **定期审计** - 检查 Nginx 访问日志
5. **备份数据** - 定期备份用户数据

## 📊 对比表

| 特性 | 启用访问控制 | 禁用访问控制 |
|------|-------------|-------------|
| 需要登录 | ✅ 是 | ❌ 否 |
| 用户隔离 | ✅ 是 | ❌ 否 |
| URL 泄露风险 | 🟡 中等 | 🔴 高 |
| 配置复杂度 | 🟡 中等 | 🟢 低 |
| 性能 | 🟡 稍慢（需验证） | 🟢 快 |
| 适合场景 | 公网/多用户 | 内网/单用户 |

## 🆘 常见问题

### Q: 禁用后还能看到登录页面吗？
A: 能。管理平台的登录功能不受影响，只是访问实例时不再需要验证。

### Q: 禁用后能阻止特定用户访问吗？
A: 不能。禁用后所有实例都是公开的，无法单独控制。

### Q: 如何知道谁访问了实例？
A: 可以查看 Nginx 访问日志：`sudo tail -f /var/log/nginx/access.log`

### Q: 禁用后会影响性能吗？
A: 会略微提升性能，因为省去了权限验证的开销。
